# C 大调数字谱听感训练设计

日期：2026-06-22

## 目标

训练不再以“直接考试”为主。用户已经能稳定听出高低和走势，真正需要建立的是两个能力：

- 把两个音之间的距离听成“二度、三度、五度”等音程。
- 听到一个音后，知道它在 C 大调数字谱 1-7 中的位置。

所以新的设计把练习改成四个可自由切换的训练模块：方向热身、音程对比、音高入门、综合挑战。

## 训练一：方向热身

方向热身用于保持已有能力，不作为主要难点。

题目会混合两类：

- 两个音：判断第二个音上升还是下降。
- 三个音：判断走势是升升、升降、降升、降降。

反馈仍然显示数字谱，例如 3 -> 7 或 1 -> 5 -> 3。

## 训练二：音程对比

音程不再直接七选一。每题只比较两个音程，让用户先听出距离差异。

训练顺序按小步推进：

- 二度 vs 三度
- 三度 vs 五度
- 二度 vs 五度
- 三度 vs 四度
- 五度 vs 八度

每题只显示当前两个选项。比如这一题只需要判断“二度 / 三度”，下一组再换成“三度 / 五度”。这样能先建立“近、远、跳跃”的听感，再逐渐扩大识别范围。

## 训练三：音高入门

音高不再凭空猜，也不一开始就三选一。程序先播放 1，再播放目标音，让用户听目标音相对 1 的位置。

训练顺序：

- 前 3 题是听熟卡片：1、3、5。用户只点“听懂了”，不判断对错。
- 接着做二选一：1 vs 5、1 vs 3、3 vs 5。
- 最后才做三选一：1、3、5。

如果答错，程序会提示“你选了什么，正确是什么”，并自动播放对照：1 -> 你选的音 -> 1 -> 正确音。这样用户不是只知道错了，而是能立刻听差别。

这样用户先建立“1 是家，3 和 5 很稳定”的感觉，再开始判断。

## 训练四：综合挑战

综合挑战才接近考试。它混合两类题：

- 完整音程识别：二度到八度。
- 完整数字谱音高识别：1 到 7，仍先播放 1 作参照。

这个模块用于检查前面训练是否转化成真正识别能力。

## 反馈和游戏元素

每组 10 题：

- 答对加分。
- 连续答对提高连击。
- 答错清空连击。
- 本组表现给出徽章。
- 答对播放上扬的开心音效。
- 答错播放下行的低落音效。

反馈重点不是惩罚错误，而是让用户知道“刚才听到的是哪一种距离或哪一个数字谱位置”。

## 技术结构

第一版仍是静态网页：

- `index.html`：页面结构。
- `src/styles.css`：界面样式和游戏反馈动效。
- `src/app.js`：训练流程、答题、积分、连击。
- `src/audio.js`：题目播放、参考音、对错反馈音。
- `src/stages.js`：四个训练模块的题目生成和判题。
- `src/storage.js`：本地设置和统计保存。

使用 Web Audio API 生成声音，使用 `localStorage` 保存本地练习数据。

## 验证标准

- 四个训练模块都能直接切换。
- 音程对比每题只显示两个候选音程。
- 音高入门前 3 题是听熟卡片，不做判断。
- 音高入门先做二选一，再做三选一。
- 音高答错后会自动播放对照。
- 综合挑战能混合音程和音高题。
- 答案和反馈详情只使用数字谱 1-7。
- 页面在 `http://127.0.0.1:8200/` 和公网穿透地址正常运行。
