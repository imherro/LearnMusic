# C 大调分阶段听音练习程序设计

日期：2026-06-22

## 目标

做一个循序渐进的听音练习小程序。第一版只使用 C 大调音阶，也就是 C、D、E、F、G、A、B，不加入升降号和复杂调式。练习方式按阶段解锁，让用户先建立“高低方向感”，再进入音程、三个音走势和单音音高辨别。

核心体验是：听题、选择答案、显示正确答案、记录是否正确。如果某一阶段连续完成 10 题且全部答对，就自动进入下一阶段。

## 阶段结构

### 第一阶段：两个音辨别高低

程序播放两个连续音，用户判断第二个音相对第一个音是“上升”还是“下降”。

题目规则：

- 只从 C 大调音阶中取音。
- 两个音不能相同。
- 可以设置题目方向：只出上升、只出下降、或随机。
- 如果用户选择“只出上升”或“只出下降”，这个设置主要用于定向训练；正式过关建议使用随机方向。

答题选项：

- 上升
- 下降

反馈：

- 用户点击后立即显示正确答案。
- 显示两个音名，例如 C -> E。
- 显示本题是否正确。

过关条件：

- 一组 10 题全部答对，解锁第二阶段。
- 如果本组出现错误，完成后停留在第一阶段，并提示重新练习。

### 第二阶段：两个音辨别音程

程序播放两个连续音，用户判断它们之间的音程。

第一版只使用 C 大调内音程，并优先使用“级数”作为答案，降低入门难度：

- 二度
- 三度
- 四度
- 五度
- 六度
- 七度
- 八度

例如 C -> E 是三度，D -> A 是五度。第一版不区分大三度、小三度、纯五度等具体性质；等用户熟悉后，再加入“大/小/纯/增减”的专业分类。

题目规则：

- 只从 C 大调音阶中取音。
- 支持上行、下行、随机方向。
- 两个音不能相同。
- 默认在一个舒适音域中生成，例如 C4 到 C5 或 C4 到 C6。

反馈：

- 显示正确音程。
- 显示两个音名。
- 可重听当前题。

过关条件：

- 一组 10 题全部答对，解锁第三阶段。

### 第三阶段：三个音辨别高低走势

程序播放三个连续音，用户判断三个音的走势。

这里不要求用户说出具体音名，只判断相邻两个音之间的方向变化。

答题选项：

- 升升：第二个音比第一个高，第三个音比第二个高。
- 升降：第二个音比第一个高，第三个音比第二个低。
- 降升：第二个音比第一个低，第三个音比第二个高。
- 降降：第二个音比第一个低，第三个音比第二个低。

题目规则：

- 三个音都来自 C 大调音阶。
- 相邻两个音不能相同。
- 可以允许第一个音和第三个音相同，例如 C -> E -> C，这是“升降”。
- 默认随机生成四种走势，避免某一种过多出现。

反馈：

- 显示正确走势。
- 显示三个音名，例如 C -> E -> D。
- 显示用户选择是否正确。

过关条件：

- 一组 10 题全部答对，解锁第四阶段。

### 第四阶段：直接辨别单个音高

程序播放一个音，用户判断它是哪一个音。

第一版答案只识别 C 大调音名，不要求判断具体八度：

- C
- D
- E
- F
- G
- A
- B

例如播放 C4 或 C5，答案都算 C。这样先训练相对清晰的 C 大调听感，后续可以再扩展成“C4、D4、E4”这种具体音高。

题目规则：

- 只出 C 大调七个音。
- 默认在 C4 到 B4 或 C4 到 B5 中随机。
- 可以加一个“播放基准音 C”的按钮，方便用户重新建立中心音。

反馈：

- 显示正确音名。
- 显示是否正确。
- 记录每个音的正确率。

## 用户流程

首页直接进入练习，不做介绍页。

界面上方显示四个阶段：

1. 两音高低
2. 两音音程
3. 三音走势
4. 单音音高

未解锁的阶段置灰。当前阶段高亮。用户可以回到已经解锁的阶段复练，但不能跳到未解锁阶段。

每一题的流程：

1. 点击开始或下一题。
2. 程序播放题目。
3. 用户可以重听。
4. 用户选择答案。
5. 程序显示正确答案和本题结果。
6. 用户进入下一题。

每组固定 10 题。组内显示当前进度，例如 4 / 10。全部答对后自动进入下一阶段；只要错一题，本组结束后不晋级。

## 设置

第一版设置保持克制，只保留真正有用的几项：

- 练习方向：上升、下降、随机。
- 播放速度：慢、中、快。
- 音域：低、中、高，默认中音区。
- 是否显示音名：答题前隐藏，答题后显示；未来可以增加“练习模式下提前显示”。

阶段一和阶段二使用“方向设置”。阶段三默认随机走势。阶段四不使用方向设置。

## 音频设计

使用浏览器 Web Audio API 生成声音，不依赖外部音频文件。第一版使用柔和的合成音色，保证音高清楚、起音不刺耳。

C 大调音高数据使用 MIDI 编号维护，例如：

- C4 = 60
- D4 = 62
- E4 = 64
- F4 = 65
- G4 = 67
- A4 = 69
- B4 = 71
- C5 = 72

程序内部用 MIDI 编号计算高低和音程，再把结果转换成用户能看懂的音名或级数。

## 数据与进度

本地保存这些数据：

- 已解锁阶段。
- 当前阶段的最近练习记录。
- 每个阶段的总正确率。
- 第四阶段每个音名的正确率。
- 最近一次设置。

使用浏览器 `localStorage` 保存，不需要登录，也不需要服务器。

## 技术结构

第一版做成静态网页：

- `index.html`：页面结构。
- `src/styles.css`：视觉样式。
- `src/app.js`：界面交互和流程控制。
- `src/audio.js`：声音生成与播放。
- `src/stages.js`：四个阶段的题目生成和判题。
- `src/storage.js`：进度和统计保存。

核心状态：

- 当前阶段。
- 当前题目。
- 当前 10 题组的答题记录。
- 已解锁阶段。
- 用户设置。

## 错误处理

需要处理的情况：

- 浏览器第一次播放声音前需要用户点击。
- 用户选择“只出上升”时，某些起始音太高，无法继续上升。
- 本地保存数据损坏或来自旧版本。

处理方式：

- “开始练习”按钮初始化音频。
- 生成题目时自动避开不合法音。
- 本地数据读取失败时重置进度，但不影响程序运行。

## 验证标准

第一版完成后需要确认：

- 第一阶段能正确判断上升和下降。
- 第一阶段 10 题全对后能进入第二阶段。
- 第二阶段音程级数计算正确。
- 第三阶段四种走势都能出现，答案判断正确。
- 第四阶段能播放单音并判断 C 到 B。
- 刷新页面后已解锁阶段仍然保留。
- 本地浏览器中可以连续完成完整四阶段流程。

## 后续扩展

第一版稳定后，可以继续加：

- 大调/小调切换。
- 更多调号。
- 第二阶段细分为大二度、小三度、纯五度等专业音程。
- 和声音程，也就是两个音同时响。
- 五线谱识别。
- 唱准检测。
