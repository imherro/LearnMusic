# 音程练习程序设计

日期：2026-06-22

## 目标

做一个可以持续练习音程听辨的小程序。第一版以“听两个音，判断是什么音程”为核心，不先做复杂的乐谱、录音识别或课程系统。这样可以最快形成可用闭环：听题、作答、反馈、复盘。

## 第一版范围

第一版默认练习“旋律音程听辨”：

- 程序播放两个连续音。
- 用户从选项中选择音程名称。
- 程序立即显示对错，并允许重听。
- 支持上行、下行、随机方向。
- 支持选择练习范围，例如小二度到纯八度。
- 显示本轮正确率、总题数、连续答对数。
- 本地记录每个音程的正确率，用于后续错题练习。

暂不做的内容：

- 不做账号系统。
- 不做服务端。
- 不做真实钢琴采样库。
- 不做麦克风唱准检测。
- 不做五线谱识谱练习。

这些能力可以作为后续版本扩展。

## 用户体验

首页就是练习界面，不做宣传页。界面分成四块：

1. 练习控制区：开始、播放、重听、下一题。
2. 答题区：音程按钮，例如小二度、大二度、小三度、大三度、纯四度、纯五度、小六度、大六度、小七度、大七度、纯八度。
3. 设置区：方向、音域、音程范围、播放速度。
4. 统计区：当前正确率、错得最多的音程、最近答题记录。

答题流程尽量短：开始后自动播放，用户点答案，看到结果，然后可以直接进入下一题。

## 音频设计

使用浏览器自带的 Web Audio API 生成声音。第一版可以先用柔和的正弦波或简单合成音，避免引入大型音频素材。

题目生成逻辑：

1. 从允许的起始音范围中随机选一个 MIDI 音高。
2. 从用户选择的音程范围中随机选一个音程。
3. 根据方向计算第二个音。
4. 如果第二个音超出安全音域，就重新生成。
5. 播放两个音，旋律模式下依次播放，和声模式后续再加入。

音程数据用半音数表示，例如：

- 小二度：1
- 大二度：2
- 小三度：3
- 大三度：4
- 纯四度：5
- 增四度/减五度：6
- 纯五度：7
- 小六度：8
- 大六度：9
- 小七度：10
- 大七度：11
- 纯八度：12

## 技术结构

建议做成静态网页项目：

- `index.html`：页面结构。
- `src/styles.css`：界面样式。
- `src/app.js`：应用入口和界面事件。
- `src/audio.js`：声音生成和播放。
- `src/exercise.js`：题目生成、答案判断。
- `src/storage.js`：本地统计保存。

状态数据包括：

- 当前题目：起始音、目标音、音程、方向。
- 当前设置：音程范围、方向、播放速度、音域。
- 当前统计：总题数、正确数、每个音程的答题记录。

本地保存使用 `localStorage`，格式保持简单，方便以后迁移。

## 错误处理

需要处理几个常见情况：

- 浏览器第一次播放声音可能需要用户点击后才允许播放。
- 用户设置的音域太窄，可能生成不出合法题目。
- 本地统计数据可能因为旧版本格式不一致而读取失败。

处理方式：

- 第一次进入时让“开始练习”按钮触发音频初始化。
- 如果设置组合无法生成题目，提示用户扩大音域或减少音程范围。
- 读取统计失败时重置统计，不影响练习。

## 测试重点

第一版验证不需要复杂测试框架，但要覆盖关键逻辑：

- 音程半音数计算正确。
- 上行和下行题目不会超出设定音域。
- 答案判断不会把等音名或方向误算进去。
- 本地统计能保存和恢复。
- 浏览器中开始、重听、答题、下一题流程顺畅。

实现完成后，用本地浏览器实际听几轮，确认声音、节奏和反馈都能正常工作。

## 实施步骤

1. 搭好静态网页结构。
2. 实现音频播放。
3. 实现题目生成和答案判断。
4. 做出基础练习界面。
5. 加入设置和统计。
6. 做本地验证并提交到 GitHub。

第一版完成标准：用户能连续完成至少 20 道音程听辨题，并看到正确率和错题分布。
